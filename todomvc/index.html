TodoMVC is a common set of JavaScript examples for MVC frameworks like JavaScriptMVC, Backbone, Spine and other libraries.

 

Why Use It?

Now more-so than ever, developers are interested in learning how to effectively apply the MVC architecture pattern to their JavaScript projects. The problem most devs experience when starting this process for the first time is trying to figure out which client-side MVC tool they should be using - JavaScriptMVC? Backbone? Spine? Sammy?. The list of solutions goes on.

In addition to good documentation, libraries should ideally also be accompanied with a good set of examples. Some of them do a great job of this whilst others could use a little work.

TodoMVC is a simple idea - the same sample application(s), built in each of the major JS MVC libraries so that you can easily find out how to:

  • Solve a common goal
  • Learn how solutions approach achieving a goal differently
  • Easily learn how to use the library in the process

Where a project already offers a particular example, what TodoMVC does is ensure that there's consistency across cross-browser compatibility, features, UI and setup information.

 

Getting Started With TodoMVC

Getting started with TodoMVC is quite straight-forward. You can either download the current stable release from the Downloads section in the sidebar or alternatively, clone the repo using the below information. 

					git clone git://github.com/addyosmani/todomvc

The structure of the download package is quite easy to follow: each application/example will have it's own folder, with tool-specific implementations stored in a dedicated sub-folder. For example, todo-example/JavaScriptMVC contains the JMVC version of the todo app.

Note: As TodoMVC aims to be completely self-contained, your download will include the libraries used for example applications and any of their dependancies as well. For libraries like Backbone and Spine this won't be an issue at all, but bare in mind some of the larger solutions may add to the size of the package.

 

About

When I first began exploring solutions that would help me use the MVC architecture pattern in my JavaScript apps, I found myself lost in a sea of choice. Many tools appeared to offer exactly what I needed, but even once I'd read their documentation and played around with their code, I still found it tricky to decide just which solution felt right for the job.

It wasn't until I began playing around with concepts like 'todo' app - a simple list-based application tool and community members had been trying to port over from one framework to others - that I really found myself understanding how they differed in implementation when it actually came around to solving problems.

The problem was, some ported implementations of the app were scattered across forums or news-groups and the developers behind the apps didn't always maintain them or patch bugs beyond the initial release.

That's really where TodoMVC came about. It's a simple idea in practice - a project aimed at providing the same application across multiple MVC frameworks starting off with the 'todo' app. Implementations will however be cross-browser compatible, self-contained (meaning you can play with them right out of the box) and I'll be working to expand the pack to cover frameworks like SproutCore and Sammy as well.

I would like to thank the list of contributors to the project, many of who worked on the initial ports of the todo-app over to other frameworks for their work. Without it this project would have taken a lot longer to get out, so thanks for your time and efforts!.
